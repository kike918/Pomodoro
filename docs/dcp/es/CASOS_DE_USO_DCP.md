# Casos de Uso DCP

## Objetivo

Definir escenarios concretos de trabajo donde Super Productivity puede agregar valor dentro de Digital Consulting Plus y su ecosistema operativo.

## 1. Normia

### Contexto
Normia es una iniciativa activa enfocada en cumplimiento operativo, procesos de inocuidad alimentaria, gestión de calidad y flujos de acciones correctivas.

### Casos candidatos
- Ejecutar tareas diarias del Sprint 0 de levantamiento.
- Medir tiempo de discovery, arquitectura, documentación, prototipo y validación.
- Planear personalmente trabajo vinculado a GitHub Issues.
- Ejecutar sesiones de foco para definición y desarrollo de producto.
- Generar resúmenes semanales por sprint.

## 2. MicroPOS

### Contexto
MicroPOS se diseña como arquitectura POS ligera, edge-capable y offline-first para pequeños negocios y entornos con conectividad limitada.

### Casos candidatos
- Seguir tareas de investigación de arquitectura.
- Separar trabajo de laboratorio de trabajo de producto.
- Timeboxear pruebas con Raspberry Pi y experimentos de sincronización offline.
- Vincular ejecución con GitHub Issues y decisiones de arquitectura.
- Generar resúmenes semanales de avances, bloqueos y próximos experimentos.

## 3. FalconCDT

### Contexto
FalconCDT incluye trabajo real de plataforma B2B de engagement, despliegues white-label y seguimiento operativo.

### Casos candidatos
- Planear trabajo de ingeniería importado desde GitHub.
- Registrar tiempo del producto core frente a trabajo específico por cliente.
- Diferenciar mantenimiento, features, QA y entrega comercial.
- Apoyar preparación de releases y planeación diaria.

## 4. DCP Consulting

### Casos candidatos
- Registrar tiempo de entrega por cliente.
- Separar discovery, implementación, soporte y documentación.
- Generar resúmenes semanales de servicio.
- Detectar esfuerzo oculto no reflejado en facturación o propuestas.
- Mejorar estimaciones de futuras ofertas comerciales.

## 5. Seguimiento Comercial

### Casos candidatos
- Preparar propuestas.
- Planear seguimientos.
- Medir esfuerzo de preparación comercial.
- Crear recordatorios personales conectados al flujo comercial.
- Alimentar resúmenes comerciales semanales mediante automatización externa.

## 6. SipDolce Labs

### Casos candidatos
- Registrar tiempo de experimentos.
- Separar trabajo de procesos de café de hardware, IoT y software.
- Planear sesiones de laboratorio.
- Medir esfuerzo relacionado con trazabilidad, etiquetado, sensores y validación de equipos.

## Modelo común de clasificación

Taxonomía recomendada de proyectos y etiquetas:

```text
Producto / Cliente
  ├── Normia
  ├── MicroPOS
  ├── FalconCDT
  ├── DCP Consulting
  ├── Comercial
  └── SipDolce Labs

Frente de trabajo
  ├── Discovery
  ├── Arquitectura
  ├── Desarrollo
  ├── QA
  ├── Documentación
  ├── Operaciones
  ├── Investigación
  └── Comercial
```

## Principio

Super Productivity debe ayudar a ejecutar trabajo. No debe convertirse en fuente de verdad para gobierno compartido de producto, decisiones de arquitectura o conocimiento empresarial.
