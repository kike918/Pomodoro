# DCP Use Cases

## Objective

Define concrete work scenarios where Super Productivity may add value inside Digital Consulting Plus and its operating ecosystem.

## 1. Normia

### Context
Normia is an active product initiative focused on operational compliance, food-safety processes, quality management and corrective-action workflows.

### Candidate use cases
- Daily execution of Sprint 0 discovery tasks.
- Time tracking by discovery, architecture, documentation, prototype and validation.
- Personal planning linked to GitHub Issues.
- Focus sessions for product definition and development.
- Weekly work summaries by sprint.

## 2. MicroPOS

### Context
MicroPOS is being designed as a lightweight, edge-capable and offline-first POS architecture for small businesses and low-connectivity environments.

### Candidate use cases
- Track architecture research tasks.
- Separate lab work from product work.
- Timebox Raspberry Pi and offline-sync experiments.
- Link execution tasks with GitHub Issues and architecture decisions.
- Produce weekly summaries of progress, blockers and next experiments.

## 3. FalconCDT

### Context
FalconCDT includes real B2B engagement platform work, white-label deployments and operational follow-up.

### Candidate use cases
- Plan engineering work imported from GitHub.
- Track time spent on core product vs. client-specific deployment operations.
- Distinguish maintenance, feature work, QA and commercial delivery.
- Support release preparation and daily focus planning.

## 4. DCP Consulting

### Candidate use cases
- Track consulting delivery time by client.
- Separate discovery, implementation, support and documentation.
- Produce weekly service-delivery summaries.
- Detect hidden effort not reflected in billing or proposals.
- Improve estimation for future commercial offers.

## 5. Commercial Follow-up

### Candidate use cases
- Prepare proposals.
- Plan follow-ups.
- Track proposal preparation effort.
- Create personal reminders connected to commercial workflows.
- Feed weekly commercial review summaries through external automation.

## 6. SipDolce Labs

### Candidate use cases
- Track experiment time.
- Separate coffee process work from hardware, IoT and software work.
- Plan lab sessions.
- Record effort related to traceability, labeling, sensors and equipment validation.

## Common classification model

Recommended project or tag taxonomy:

```text
Product / Client
  ├── Normia
  ├── MicroPOS
  ├── FalconCDT
  ├── DCP Consulting
  ├── Commercial
  └── SipDolce Labs

Workstream
  ├── Discovery
  ├── Architecture
  ├── Development
  ├── QA
  ├── Documentation
  ├── Operations
  ├── Research
  └── Commercial
```

## Principle

Super Productivity should help execute work. It should not become the source of truth for shared product governance, architecture decisions or company knowledge.
